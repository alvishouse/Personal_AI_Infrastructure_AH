# Brand Voice Consistency Workflow

**Focused review of brand voice and tone alignment.**

Use this workflow when you need to verify voice consistency without full editorial review.

---

## Purpose

Rapid assessment of:
- Tone alignment with brand voice
- Vocabulary and language choices
- Perspective (1st/2nd/3rd person usage)
- Personality consistency
- Example and metaphor quality

**Time investment:** Fast, focused on voice only.

---

## When to Use

✅ **Use Voice Check when:**
- User asks "does this sound like me?"
- Content written by someone else needs voice alignment
- Testing new content types for voice fit
- Multiple authors contributing, need consistency
- Quick voice audit before publish

❌ **Don't use Voice Check for:**
- First comprehensive review (use FullEdit)
- Grammar and technical editing (use QuickPolish)
- SEO optimization (use SEOReview)

---

## 🚨 VOICE CHECK STEPS

```
INPUT: Blog content (any draft stage)
     ↓
[1] REFERENCE: Load target voice profile
     ↓
[2] SAMPLE: Read representative sections
     ↓
[3] ASSESS: Apply voice framework
     ↓
[4] REPORT: Voice consistency score
     ↓
OUTPUT: Alignment verdict + adjustments
```

---

## Step 1: Reference Target Voice

**Target Voice Profile:**

```
VOICE ATTRIBUTES:
- Intellectual yet accessible
- Conversational authority (expert friend, not guru)
- Curiosity-driven (asks questions, explores ideas)
- Pattern-recognizing (connects dots across domains)
- Systems-thinking (sees structures and relationships)
- Slightly irreverent (questions orthodoxy, not rude)
- Practical idealism (grounded vision)

TONE:
- Thoughtful but not ponderous
- Confident but not arrogant
- Engaging but not breathless
- Serious but not stuffy

VOCABULARY:
- Precise technical terms when needed
- Plain English preferred
- Metaphors from physical world (mechanical, architectural)
- Avoids: buzzwords, hype, marketing-speak
- Uses: specific examples, concrete data, visual language

PERSPECTIVE:
- "We" for collaborative exploration
- "You" for reader-focused benefit
- "I" sparingly (for personal experience only)
- Rarely 3rd person omniscient

SENTENCE STRUCTURE:
- Varied length (mix short punchy with longer exploratory)
- Active voice preferred (>90%)
- Questions used to drive curiosity
- Lists and bullets for clarity

EXAMPLES & METAPHORS:
- Physical/mechanical (levers, gears, bridges)
- Visual/architectural (blueprints, scaffolding)
- Everyday objects (napkins, whiteboards, tools)
- Avoids: sports metaphors, war metaphors, tired business clichés
```

See `templates/StyleGuide.md` for complete reference.

## Step 2: Sample Key Sections

**Read these sections carefully:**

1. **Opening paragraph** - Sets tone for entire piece
2. **First subheading section** - Typical voice throughout
3. **Example or case study** - How examples are presented
4. **Conclusion** - Final impression on reader

Don't need to read entire piece - sample is sufficient for voice check.

## Step 3: Apply Voice Consistency Framework

From `EditingFrameworks.md`:

```
VOICE CONSISTENCY REPORT
═══════════════════════════════════════════════════════════

CONTENT: [Title/section being reviewed]

ASSESSMENT:

□ TONE ALIGNMENT: [PASS/ADJUST]
  Expected: Thoughtful, analytical, engaging
  Found: [Your assessment]

  Issues (if any):
  - [Specific examples where tone is off]
  - [Sections that feel too X or not enough Y]

□ VOCABULARY MATCH: [PASS/ADJUST]
  Expected: Precise technical + plain English
  Found: [Your assessment]

  Issues (if any):
  - [Buzzwords or hype language: "revolutionary", "game-changing"]
  - [Unnecessary jargon without explanation]
  - [Missing concrete examples]

□ PERSPECTIVE: [PASS/ADJUST]
  Expected: "We" (collaborative) + "You" (reader-focused)
  Found: [Your assessment]

  Issues (if any):
  - [Overuse of "I" or "me"]
  - [Too distant/academic (all 3rd person)]
  - [Inconsistent POV shifts]

□ PERSONALITY: [PASS/ADJUST]
  Expected: Curious expert, not preachy guru
  Found: [Your assessment]

  Issues (if any):
  - [Sounds too authoritative/preachy]
  - [Lacks personality/sounds generic]
  - [Wrong personality (too casual, too formal)]

□ EXAMPLES & METAPHORS: [PASS/ADJUST]
  Expected: Concrete, visual, relatable (mechanical/architectural)
  Found: [Your assessment]

  Issues (if any):
  - [Abstract examples that don't land]
  - [Wrong metaphor style (sports/war instead of mechanical)]
  - [Missing metaphors where helpful]

═══════════════════════════════════════════════════════════
OVERALL VOICE SCORE: [X/10]

VERDICT: [OFF-BRAND / INCONSISTENT / ALIGNED / EXEMPLARY]
```

## Step 4: Provide Voice Adjustments

**When voice issues found, provide specific examples:**

### Format for Adjustments:

```
VOICE ADJUSTMENTS NEEDED
─────────────────────────────────────────────────────────────

1. TONE ADJUSTMENT - [Location in draft]

   Current: "[Exact quote from draft]"
   Issue: [Too formal / too casual / too marketing-y / etc.]
   Revised: "[Your voice-aligned rewrite]"

2. VOCABULARY FIX - [Location in draft]

   Current: "[Exact quote with problematic words]"
   Issue: [Buzzword / jargon / vague language]
   Revised: "[Concrete, plain English version]"

3. PERSPECTIVE SHIFT - [Location in draft]

   Current: "[Exact quote]"
   Issue: [Wrong POV / too distant / too personal]
   Revised: "[Correct perspective version]"

[Continue for all voice breaks]
```

## Common Voice Issues & Fixes

### Issue: Too formal/academic

```
❌ "One must consider the implications of artificial intelligence deployment"
✅ "Think about what happens when you deploy AI"

❌ "The aforementioned framework provides..."
✅ "This framework helps you..."
```

### Issue: Too marketing-y/hyped

```
❌ "Revolutionary AI that will transform everything!"
✅ "AI that changes how teams work - here's how"

❌ "Game-changing breakthrough"
✅ "A different approach worth trying"
```

### Issue: Wrong POV

```
❌ "I think companies should..."
✅ "Companies that amplify humans (not replace them) see..."

❌ "They need to understand..."
✅ "You need to understand..." or "We need to understand..."
```

### Issue: Missing personality

```
❌ "AI systems can improve efficiency."
✅ "Here's the thing about AI: it's a terrible replacement but an excellent amplifier."

❌ "Consider these factors:"
✅ "Before you automate everything, ask yourself:"
```

### Issue: Wrong metaphor style

```
❌ "Winning the AI race" (sports)
✅ "Building AI leverage" (mechanical)

❌ "AI warfare strategy" (war)
✅ "AI architecture decisions" (architectural)

❌ "Slam dunk AI solution" (cliché + sports)
✅ "AI as a force multiplier, not a replacement" (mechanical + specific)
```

## Voice Check Workflow Checklist

**Before starting:**
- [ ] Review target voice profile (above or StyleGuide.md)
- [ ] Sample key sections (don't need full read)

**During assessment:**
- [ ] Check tone alignment
- [ ] Evaluate vocabulary choices
- [ ] Verify perspective consistency
- [ ] Assess personality fit
- [ ] Review examples and metaphors

**Report generation:**
- [ ] Provide overall voice score
- [ ] List specific issues with examples
- [ ] Give concrete adjustments
- [ ] Clear verdict with next steps

---

## When to Escalate

**Voice Check reveals bigger issues:**

If voice problems are pervasive (score < 4/10), recommend:
1. **FullEdit workflow** - May indicate structural/Big Idea issues
2. **Rewrite consideration** - Sometimes easier to start fresh
3. **Voice training** - Author may need voice profile reference

---

**Voice consistency is brand consistency. Worth the focused check.**
