✕

✕
No defined handoffs — agent output goes nowhere structured
✕

The DARI Framework
D Discover
Map before you automate
A Architect
Design the handoffs
R Run
Execute with visibility
I Improve
Measure drift, not just breaks
Discover
Map the work before you automate it.
✓
Identify every human-agent boundary
✓
Document inputs, outputs, and done-right criteria
✓
Define success before you define the agent
Architect
Design the handoffs, not just the tasks.
→
Human → Agent: What does it need?
←
Agent → Human: What needs verification?
⇄
Agent → System: Guardrails before live data
Every handoff: Clear · Verifiable · Recoverable
Run
Execute with visibility.
✓
Log outputs — every run leaves a traceable record
✓
Flag deviations before they compound
✓
Designated owner — accountability doesn't disappear when the worker is a model
Improve
Measure what's drifting, not just what's broken.
⚡
Capability Drift: The model itself changes
↘
Performance Drift: Output declines, no model change
⟳
Context Drift: Business changed, agent didn't
Review cadence: Daily · Weekly · Monthly
The Bottom Line
The Positioning
DARI isn't a replacement for Lean.
It's Lean extended to the delegation problem.
Start This Week
Pick one agent process. List every handoff boundary.
Assign a human owner. Add one log checkpoint.
